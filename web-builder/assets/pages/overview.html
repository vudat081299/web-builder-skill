Bộ component CSS tối giản để build web

Một thư viện CSS + hệ thiết kế tối giản, ưu tiên trắng–đen–xám, dùng lại được — viết tay, token-based, không cần build, drop-in một file, prefix wb-* nên ghép được với mọi stack. Mạnh nhất cho web tài chính cá nhân (bảng tiền, ngân sách, hoá đơn) nhưng dùng tốt cho web bất kỳ. Mỗi nhóm nằm ở một trang riêng — chọn ở sidebar để xem đúng nhóm. Khi build web, ghép từ catalog thay vì thiết kế lại từ đầu.

Repo này là gì

Sản phẩm cuối cùng của repo = bộ skill web-builder — thứ duy nhất “ship”: một file CSS drop-in (web-builder.css) cùng SKILL.md + references/. Trang docs bạn đang xem, /wb-change và các hook không ship — chúng chỉ để dựng, review, dogfood và bảo vệ chất lượng của skill.

Cách lib hoạt động: CSS viết tay, token-based (đổi token là đổi cả hệ), không cần build, một file duy nhất, prefix wb-* nên ghép với mọi stack mà không xung đột. Ưu tiên trắng–đen–xám, có dark mode. Khi build web thì ráp lại từ catalog thay vì thiết kế lại — nguyên tắc reuse-first.

Ba lớp — mỗi lớp cho ai

Skill — ship

web-builder.css + SKILL.md + references/. Cho AI dựng UI web: drop-in một file rồi ráp từ class wb-* + token. Đây là thứ đi vào app của bạn.

Docs site — không ship

index.html · app.js · pages/ (trang này). Cho người & AI tra cứu, xem thật từng component ở light/dark và verify trước khi tin.

Tooling — không ship

/wb-change · hook · validate-sync.sh. Để thêm/sửa component an toàn và giữ 6 nơi (CSS ↔ demo ↔ NAV ↔ catalog ↔ SKILL ↔ principles) không lệch nhau.

AI làm gì với repo này

Repo được thiết kế để một AI vừa dùng vừa mở rộng bộ component — không chỉ để tra cứu.

Build UI web

Ráp màn hình từ wb-* + token, bám thang màu (trắng–đen–xám trước, màu chỉ cho status). Không tự chế style.

Thêm primitive mới

Chạy /wb-change: discover → plan → implement → đồng bộ 6 nơi → verify → rà soát tái dùng → review → commit.

Update component đã có

Cùng workflow; sửa xong phải rà toàn repo tìm chỗ thay được bằng bản mới, hoặc áp biến thể/token vừa thêm.

Sync docs ↔ lib ↔ skill

Giữ components-catalog.md, SKILL.md, CSS và demo khớp nhau — lệch một nơi là skill nói dối AI kế tiếp; validate-sync.sh gác cổng khi commit.

So với lib khác

Đối chiếu phạm vi với Bootstrap (bootstrap-comparison.md) để lộ khoảng trống trong bộ component, quyết định thêm gì tiếp.
